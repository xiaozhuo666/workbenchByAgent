# Quickstart - AI 票务工作台接力体验

## 1. 目标

用最短路径验证“聊天 -> 行程卡 -> 票务页自动查询 -> 推荐可应用”的 MVP 闭环，并覆盖异常降级场景。

## 2. 前置条件

- 已完成数据库迁移并存在行程草稿表。
- 登录态可用，拥有正常聊天权限。
- 12306 MCP 查询能力在测试环境可访问（或具备可控的模拟数据）。

## 3. 核心联调路径（Happy Path）

1. 在助手输入：`帮我查明天上海到北京高铁`。  
2. 验证助手返回行程卡，包含出发地、到达地、日期与偏好。  
3. 点击 `查看结果`，进入 `/tickets?draftId=...`。  
4. 验证页面自动填充并自动触发查询。  
5. 验证结果区展示直达列表，且可切换到中转优选。  
6. 验证 AI 建议区展示最快/最便宜/最舒适三类推荐。  
7. 点击某推荐卡的应用操作，验证筛选条件变化并刷新结果。

## 4. 异常与恢复路径

### 场景 A：草稿过期
1. 使用过期 `draftId` 进入票务页。
2. 预期：页面显示草稿过期提示，并提供“重新生成/手动修改后重查”。

### 场景 B：上游查询部分失败
1. 模拟上游返回 partial。
2. 预期：列表可展示已返回数据，并提示“结果可能不完整”。

### 场景 C：推荐接口失败
1. 模拟推荐服务失败。
2. 预期：基础结果列表与筛选能力可正常使用，推荐区展示重试或失败提示。

## 5. 回归检查

- 聊天普通问答与历史会话功能不回退；
- 原首页核心能力（待办/日程）不受影响；
- 登录鉴权与路由守卫行为保持一致；
- 新增票务页在 375/768/1024/1440 断点下可正常浏览和操作。
