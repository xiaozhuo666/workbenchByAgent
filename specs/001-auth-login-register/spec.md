# Feature Specification: 登录注册与会话管理

**Feature Branch**: `001-auth-login-register`  
**Created**: 2026-03-01  
**Status**: MVP Completed（US1 注册/登录 + US2 登录态保持与退出 已交付并验证）  
**Input**: User description: "请为智能工作台助手创建第一个功能规格：实现基本的登录/注册能力，范围包含注册、登录、会话保持、登出与未登录拦截。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 账号注册与登录 (Priority: P1) ✅ 已实现

作为首次使用工作台的用户，我可以创建账号并使用账号登录系统，成功进入工作台首页。

**Why this priority**: 登录与注册是进入工作台所有后续能力的前置条件，未完成则其他模块无法使用。

**Independent Test**: 使用一个新用户名完成注册后立刻登录，系统允许进入工作台首页并提示登录成功。

**Acceptance Scenarios**:

1. **Given** 用户未登录，**When** 访问工作台任意受保护页面，**Then** 系统跳转到登录页或展示登录入口。
2. **Given** 用户位于登录页并输入正确账号凭据，**When** 提交登录，**Then** 系统登录成功并进入工作台首页，同时建立可识别当前用户的登录态。
3. **Given** 用户位于注册页并输入新用户名与符合规则的密码，**When** 提交注册，**Then** 系统完成唯一性校验并创建账号记录。

---

### User Story 2 - 登录态保持与退出 (Priority: P1) ✅ 已实现

作为已登录用户，我在刷新页面后无需重复登录；当我主动退出后，需要重新登录才能访问工作台功能。

**Why this priority**: 稳定的登录态是日常使用体验的关键，退出后回收访问权限是基础安全要求。

**Independent Test**: 登录后刷新页面仍保持在已登录状态；执行登出后再次访问受保护页面会被拦截并返回登录入口。

**Acceptance Scenarios**:

1. **Given** 用户已登录且登录态有效，**When** 用户刷新页面或重新打开应用，**Then** 系统保持已登录状态并允许继续访问工作台。
2. **Given** 用户已登录，**When** 用户执行登出操作，**Then** 系统清除客户端与服务端登录态并返回登录页。

**实现验证（2026-03-01）**: 后端 `GET /api/auth/me`、`POST /api/auth/logout` 已实现；前端启动时调用 `restoreSession()` 拉取 `/me` 恢复用户信息，`ProtectedRoute` 无 token 时重定向至 `/auth`，`AuthLogoutButton` 调用登出 API 并清理本地存储，`httpClient` 在 401/403 时自动清除登录态；契约测试与前端守卫测试已覆盖。

---

### Edge Cases

- 注册时用户名已存在：系统明确提示“用户名已被占用”，且不会创建重复账号。
- 注册时密码强度不达标：系统提示强度要求并拒绝提交。
- 登录态已过期：访问受保护页面时自动要求重新登录。
- 后端服务异常：系统展示可理解的错误信息，并允许用户重试。
- 数据存储异常：系统展示失败原因的用户可读提示，不暴露敏感内部信息。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-A01**: 系统 MUST 提供登录入口，用户可使用账号标识与密码发起登录，校验成功后建立登录态。
- **FR-A02**: 系统 MUST 提供注册入口，注册时必须校验账号唯一性，并在通过后创建新用户记录。
- **FR-A03**: 系统 MUST 在登录态有效期内保持用户会话，使用户刷新页面后仍可访问已授权功能。
- **FR-A04**: 系统 MUST 提供登出能力，登出后必须使当前登录态失效。
- **FR-A05**: 系统 MUST 对未登录访问进行拦截，并将用户引导至登录入口。

### Key Entities *(include if feature involves data)*

- **User**: 表示系统账号主体，包含唯一账号标识、认证凭据摘要、基础资料与创建时间。
- **Session**: 表示一次登录会话，包含会话标识、关联用户、创建时间、过期时间与状态。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-00**: 目标用户可在 2 分钟内完成“注册并登录”，并成功进入工作台首页。
- **SC-01**: 在有效登录态下，连续刷新页面 3 次后仍保持登录成功率达到 100%。
- **SC-02**: 用户执行登出后，再次访问受保护页面的拦截成功率达到 100%。
- **SC-03**: 对于“用户名重复、密码不合规、会话过期、服务异常”四类场景，系统均提供可理解错误提示，验收用例通过率达到 100%。

## Scope Boundaries

- 本功能仅包含：注册、登录、会话保持、登出、未登录拦截。
- 本功能不包含：忘记密码、邮箱验证、第三方登录。

## Assumptions & Dependencies

- 本规格遵循项目宪章中的既定系统与数据约束。
- 本功能面向单一应用内账号体系，不包含跨系统单点登录。
- 受保护页面至少包含工作台首页及其核心入口。
