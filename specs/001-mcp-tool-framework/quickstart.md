# Quickstart - MCP Tool Adapter

## 1. 前置条件

- 已完成数据库初始化与后端依赖安装。
- 可正常访问现有 `/api/ai/chat` 接口。
- 准备好模型密钥与数据库连接配置。

## 2. 配置环境变量（建议）

在后端运行环境中新增：

- `MCP_ENABLED=false`
- `MCP_TOOL_WHITELIST=tool.weather.query`
- `MCP_MAX_CALL_ROUNDS=3`
- `MCP_TOOL_TIMEOUT_MS=3000`
- `MCP_TOOL_MAX_RETRIES=1`
- `MCP_RETRY_BACKOFF_MS=200`

已有变量继续沿用：

- `DASHSCOPE_API_KEY`
- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`
- `JWT_SECRET`, `CORS_ORIGIN`

## 3. 开发执行步骤

### Step A：实现工具编排基础能力

1. 新增 `toolRegistry`、`toolExecutor`、`chatOrchestrator` 模块。
2. 在 `ai.service.js` 中接入编排入口，保留旧路径。
3. 在 `ai.controller.js` 通过开关控制新旧路径。

**验证方式**:
- MCP 关闭：`/api/ai/chat` 行为与旧版一致。
- MCP 开启：命中白名单工具时可走调用闭环并得到回复。

### Step B：补全可靠性与可观测

1. 增加超时、重试、白名单校验。
2. 增加工具调用日志落库与失败不阻断机制。
3. 增加关键指标采样（成功率、失败率、耗时、降级率）。

**验证方式**:
- 人工注入超时/异常，接口仍返回可用内容。
- 日志表中可查到成功与失败记录，字段完整。

### Step C：灰度与上线

1. 预发环境开启 MCP，执行全量验收。
2. 生产先小流量灰度，观察指标。
3. 超阈值立即关闭开关回滚。

**验证方式**:
- 关键验收场景全部通过（工具成功/超时/异常/降级成功）。
- 回滚后核心接口恢复历史基线。

### Step D：最小 MCP 开关页

1. 新增管理页面（工具列表 + 开关按钮）。
2. 新增前端 API 封装用于查询/更新开关状态。
3. 新增后端管理接口并接入管理员权限控制。

**验证方式**:
- 页面可正确展示每个工具启停状态。
- 切换开关后 3 秒内生效，且聊天调用行为同步变化。
- 页面新增不影响首页与原有核心交互流程。

## 4. 测试清单（最小可执行）

- 单元测试：
  - registry 注册与白名单判定
  - executor 超时与重试行为
  - loop 终止与降级策略
- 集成测试：
  - `/api/ai/chat` 开关关闭兼容
  - `/api/ai/chat` 开关开启成功链路
  - 超时与异常自动降级
  - `/api/ai/mcp/tools` 查询与更新开关接口
- 回归测试：
  - 登录、待办、日程、AI 对话
  - 最小 MCP 开关页

## 5. 回滚指令与判定

- **回滚动作**: 设置 `MCP_ENABLED=false` 并重启后端服务。
- **触发条件**:
  - 工具失败率持续高于阈值
  - 主聊天接口错误率异常上升
  - 平均耗时超出可接受范围
  - 开关管理接口异常导致误启/误关

回滚后继续保留日志以便复盘，并在下一轮灰度前修复问题。
