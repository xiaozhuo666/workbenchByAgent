# Phase 0 Research - MCP Tool Adapter

## 决策 1：采用“编排层 + 注册中心 + 执行器”三段式

- **Decision**: 在 `ai` 模块内部拆分为 `toolRegistry`、`toolExecutor`、`chatOrchestrator` 三个责任层。
- **Rationale**: 便于后续平滑接入多个 MCP 工具，且可在不改变对外 API 的前提下替换内部策略。
- **Alternatives considered**:
  - 将工具逻辑直接写入 `ai.service.js`：短期快，但后续扩展和测试成本高。
  - 每个工具单独嵌入 controller：会导致业务流程分散，难以统一超时/重试/降级。

## 决策 2：工具调用策略采用“有限轮次 + 每轮超时预算”

- **Decision**: 使用可配置 `maxRounds` 限制循环轮次，并为每次执行设置 timeout 与重试上限。
- **Rationale**: 能有效防止死循环和长尾请求，符合“失败自动降级”要求。
- **Alternatives considered**:
  - 不限制轮次：存在无限循环风险。
  - 全局单次超时不分轮：无法精细控制各工具调用风险。

## 决策 3：白名单策略采用“默认拒绝，显式放行”

- **Decision**: 未注册或未在白名单中的工具一律拒绝执行；本期仅允许 `read` 能力标签工具。
- **Rationale**: 满足安全边界和“禁止自动写操作”约束，降低误调用高风险能力概率。
- **Alternatives considered**:
  - 默认允许 + 黑名单拦截：维护困难且易漏配。
  - 仅靠提示词约束：不可审计，且防护强度不足。

## 决策 4：日志模型采用“调用事件 + 会话聚合”双层记录

- **Decision**: 每次调用记录明细事件（耗时、状态、错误），并按会话聚合统计（成功/失败/降级）。
- **Rationale**: 既能支持排障追踪，也能支持灰度阶段指标监控。
- **Alternatives considered**:
  - 仅记录最终结果：无法定位中间调用失败点。
  - 全量记录参数正文：存在隐私与存储膨胀风险。

## 决策 5：降级策略采用“工具链失败即回退纯模型回复”

- **Decision**: 工具超时、重试耗尽、参数校验失败、结果回灌失败都触发统一降级分支。
- **Rationale**: 对用户体验最稳定，且满足“工具失败必须自动降级”的硬性要求。
- **Alternatives considered**:
  - 失败直接返回错误：会破坏现有聊天可用性。
  - 部分错误不降级：会产生不一致行为，增加前端兼容成本。

## 决策 6：灰度发布采用“总开关 + 工具白名单 + 阈值回滚”

- **Decision**: 通过 `MCP_ENABLED` 控制总开关，结合白名单与指标阈值逐步放量。
- **Rationale**: 实施简单、风险可控、可快速回滚，不影响现有核心功能。
- **Alternatives considered**:
  - 代码分支发布：切换成本高且回滚慢。
  - 一次性全量开启：风险集中，难以在早期发现问题。

## 决策 7：本期纳入“最小 MCP 开关页”

- **Decision**: 本期新增最小管理页面，仅提供“工具列表 + 单工具启停”能力，不纳入权限矩阵、审计筛选、统计报表。
- **Rationale**: 该能力直接支撑灰度和快速回滚，投入可控，同时不改动前端核心业务交互路径。
- **Alternatives considered**:
  - 本期不做页面，仅靠环境变量：线上操作链路长，灵活性不足。
  - 直接做完整后台：范围过大，影响交付节奏和回归风险。

## 决策 8：开关状态采用“数据库持久化 + 运行时缓存短 TTL”

- **Decision**: 工具启停状态写入 MySQL，服务端以短 TTL 缓存读取，提高稳定性与生效速度。
- **Rationale**: 避免频繁查询数据库带来的开销，同时保证开关修改在可控时间窗口内生效。
- **Alternatives considered**:
  - 仅内存状态：重启丢失，不利于运维一致性。
  - 仅数据库实时读取：并发下增加数据库压力。

## 决策 9：开关管理接口沿用现有鉴权并限制管理员操作

- **Decision**: 开关相关接口复用现有登录鉴权，额外加管理员权限约束。
- **Rationale**: 避免未授权用户误操作工具状态，降低线上风险。
- **Alternatives considered**:
  - 所有登录用户可改：存在误操作风险。
  - 新建独立鉴权体系：复杂度高且不必要。
