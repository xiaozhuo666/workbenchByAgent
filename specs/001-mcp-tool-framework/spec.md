# Feature Specification: 个人生活助手 MCP 基础框架

**Feature Branch**: `001-mcp-tool-framework`  
**Created**: 2026-03-04  
**Status**: Draft  
**Input**: User description: "为个人生活助手项目建立 MCP 基础框架，覆盖工具注册、执行、循环调用、失败降级与日志，且不改变现有对外行为。"

## Clarifications

### Session 2026-03-04

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 聊天中安全调用工具 (Priority: P1)

作为终端用户，我在聊天中提出需要外部能力的问题时，系统可在受控范围内自动调用可用工具并返回更准确结果，同时保持现有聊天体验不变。

**Why this priority**: 这是该框架的核心价值，直接决定功能是否可用且不会影响现有主流程。

**Independent Test**: 仅启用一个可读工具并发起聊天请求，验证模型可触发工具调用、接收结果并产出最终回复，且接口响应结构保持兼容。

**Acceptance Scenarios**:

1. **Given** 请求命中白名单工具能力，**When** 用户发起聊天，**Then** 系统在限定轮次内完成"决策-调用-回灌-回复"并返回最终答案。
2. **Given** 请求不需要工具，**When** 用户发起聊天，**Then** 系统直接返回模型回复且不增加额外副作用。

---

### User Story 2 - 工具失败自动降级 (Priority: P2)

作为终端用户，当外部工具超时或异常时，我仍然能收到可读、可用的回复，而不是接口错误或长时间等待。

**Why this priority**: 降级能力决定可用性与稳定性，避免工具问题放大为用户侧故障。

**Independent Test**: 模拟工具超时、重试失败、执行异常三类场景，验证系统均能转为纯模型回答并保持成功返回。

**Acceptance Scenarios**:

1. **Given** 工具执行超时且超过重试上限，**When** 聊天请求继续处理，**Then** 系统自动降级为纯模型回答并完成响应。
2. **Given** 工具返回不可解析结果，**When** 回灌阶段发生失败，**Then** 系统记录失败并返回不依赖工具的备选答案。

---

### User Story 3 - 运营可观测与可审计 (Priority: P3)

作为运营或排障人员，我可以通过工具调用日志定位某次会话的调用路径、耗时和失败原因。

**Why this priority**: 该能力保障后续接入更多工具时可快速诊断问题并支持回归验证。

**Independent Test**: 在同一会话内触发成功与失败调用，核验日志字段完整、状态清晰、可按会话追溯。

**Acceptance Scenarios**:

1. **Given** 一次工具调用完成，**When** 请求结束，**Then** 日志中存在会话标识、工具名、参数摘要、耗时与状态。
2. **Given** 一次工具调用失败，**When** 请求结束，**Then** 日志中存在错误信息且可与最终降级回复关联。

---

### User Story 4 - 手动控制 MCP 开关 (Priority: P3)

作为运营或管理员，我可以在一个最小管理页面中手动启停不同 MCP 工具，以便灰度发布和快速回滚。

**Why this priority**: 该能力直接支持上线风险控制，但不属于主链路核心对话能力，优先级低于工具调用与降级。

**Independent Test**: 在不修改聊天接口结构的前提下，通过页面关闭某工具后验证该工具不再被调用，再重新开启验证可恢复调用。

**Acceptance Scenarios**:

1. **Given** 管理页面已加载工具列表，**When** 管理员关闭某 MCP 开关，**Then** 后续请求不再调用该工具。
2. **Given** 某 MCP 已关闭，**When** 管理员重新开启，**Then** 在命中条件下可恢复工具调用。

---

### Edge Cases

- 当模型连续建议调用工具导致接近死循环时，系统在达到最大轮次后停止调用并输出最终回复。
- 当模型尝试调用未在白名单中的工具时，系统拒绝执行并继续流程，不中断用户请求。
- 当工具参数缺失关键字段时，系统不执行危险调用，记录错误后走降级路径。
- 当日志写入短暂失败时，不阻断主请求；主流程仍应返回用户可用答案。
- 当管理员误关闭关键工具时，系统仍可通过纯模型降级路径保证聊天可用。

## Requirements *(mandatory)*

### 业务流程与时序

1. 用户发起聊天请求。  
2. 系统请求模型进行回复决策（可含工具调用意图）。  
3. 若模型选择工具调用，系统执行白名单校验与参数校验。  
4. 系统调用工具执行器，按超时与重试策略获取结果。  
5. 系统将工具结果回灌给模型，生成下一步决策或最终回复。  
6. 达到最大调用轮次或得到最终文本后结束循环并返回响应。  
7. 任一步骤若工具链失败，系统自动降级为纯模型回答并返回。  
8. 全流程输出工具调用日志用于审计与排障。

### Functional Requirements

- **FR-001**: 系统必须提供工具注册中心，用于登记、查询和启停可用工具元数据。
- **FR-002**: 系统必须仅允许执行白名单内工具，且在执行前完成工具存在性校验。
- **FR-003**: 系统必须提供统一工具执行器，支持超时控制、有限次数重试与结果标准化。
- **FR-004**: 聊天处理流程必须支持工具调用循环，并可配置最大轮次以防死循环。
- **FR-005**: 当工具调用或回灌失败时，系统必须自动切换为纯模型回答，且不向用户暴露内部异常堆栈。
- **FR-006**: 新增工具框架后，现有登录、待办、日程、AI 对话既有接口行为与响应结构必须保持兼容。
- **FR-007**: 系统必须记录每次工具调用日志，至少包含会话标识、工具名、参数摘要、耗时、状态与错误信息。
- **FR-008**: 系统必须支持后续扩展新工具接入，不要求修改聊天对外接口契约。
- **FR-009**: 系统必须禁止高风险写操作类工具在本期上线范围内被执行。
- **FR-010**: 系统必须提供最小管理页面，支持按工具维度手动启停 MCP 开关（仅开关能力，不包含复杂权限与统计管理）。

### Non-Functional Requirements

- **NFR-001 可用性**: 当工具不可用时，用户请求仍应在可接受时长内获得可读回复。
- **NFR-002 可靠性**: 在网络抖动等临时故障下，系统通过重试与降级保证请求完成率。
- **NFR-003 安全性**: 所有外部凭据必须通过环境配置管理，不得硬编码到代码或日志。
- **NFR-004 可观测性**: 日志字段应支持按会话、工具与状态进行检索与聚合。
- **NFR-005 兼容性**: 引入框架后，既有核心业务流程不出现行为回归。
- **NFR-006 可运维性**: 工具开关变更应在一次配置刷新周期内生效，并支持快速回退到关闭状态。

### 数据与日志字段设计

- **工具定义（ToolDefinition）**: 工具标识、显示名称、用途说明、启用状态、风险等级、参数约束摘要。
- **工具执行记录（ToolExecutionRecord）**: 会话标识、消息标识、轮次序号、工具标识、参数摘要、开始时间、结束时间、耗时、执行状态、错误码、错误信息摘要。
- **会话内工具轨迹（ConversationToolTrace）**: 会话标识、累计调用次数、成功次数、失败次数、是否触发降级、最终回复类型。

### API/模块边界与接口契约

- **聊天编排模块**: 负责决策循环控制与最终回复输出；对外契约保持现有聊天接口输入输出不变。
- **工具注册中心模块**: 负责工具生命周期管理与白名单判定；对聊天编排暴露"查询可调用工具"能力。
- **工具执行器模块**: 负责参数校验、超时控制、重试、结果规范化；对聊天编排暴露"执行工具并返回结构化结果"能力。
- **日志模块**: 负责记录工具调用事件；失败时不阻断主流程。

### 验收标准与回归测试范围

- **AC-001**: 在含工具场景中，系统可在最大轮次内完成调用闭环并返回答案。
- **AC-002**: 在工具超时、异常、返回非法结果时，系统均可自动降级且接口可用。
- **AC-003**: 在未命中白名单工具时，系统拒绝执行并给出正常最终回复。
- **AC-004**: 工具调用日志字段完整率达到 100%，可按会话追踪调用链。
- **AC-005**: 现有登录、待办、日程、AI 对话关键用例回归全部通过。
- **AC-006**: 管理页面可手动启停单个 MCP；关闭后立即阻断该工具调用，开启后可恢复调用。

**回归测试范围**:
- 账号登录与鉴权主流程。
- 待办新增、查询、更新、删除主流程。
- 日程新增、查询与展示主流程。
- AI 对话在"无需工具"与"工具失败降级"两类路径的返回兼容性。

### 风险与回滚策略

- **风险 R1**: 工具循环策略配置不当导致响应时长上升。  
  **缓解**: 设置保守默认轮次并监控超时占比。  
  **回滚**: 关闭工具循环开关，恢复纯模型路径。
- **风险 R2**: 新工具接入质量不稳定影响会话体验。  
  **缓解**: 默认灰度启用，未通过验证不加入白名单。  
  **回滚**: 按工具粒度禁用并保留主流程。
- **风险 R3**: 日志量增长带来存储与检索压力。  
  **缓解**: 参数仅记录摘要并控制日志粒度。  
  **回滚**: 降低非关键字段采集频率，保留核心审计字段。

### Assumptions

- 当前用户仅通过既有聊天入口触发该能力，不新增前端大改版需求。
- 本期开关页仅提供最小化能力（工具列表 + 开关按钮），不引入复杂后台管理功能。
- 本期仅支持低风险、只读型工具能力验证，高风险交易操作继续排除在外。
- 现有会话标识机制可复用，无需改变外部接口字段定义。

### Key Entities *(include if feature involves data)*

- **ToolDefinition**: 描述可被系统调用的工具及其准入属性，用于白名单控制与扩展管理。
- **ToolExecutionRecord**: 描述单次工具调用全过程，用于审计、排障和性能分析。
- **ConversationToolTrace**: 描述单会话多轮工具调用结果，用于判断是否触发降级及评估效果。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在目标业务问答样本中，至少 95% 的请求可在一次会话内成功完成并返回可读答案（含降级路径）。
- **SC-002**: 在需要工具的样本中，至少 90% 的请求能在最大轮次限制内完成处理，不出现无限调用。
- **SC-003**: 工具调用日志关键字段（会话标识、工具名、参数摘要、耗时、状态、错误信息）完整率达到 100%。
- **SC-004**: 上线后首轮回归中，既有登录、待办、日程、AI 对话核心用例通过率达到 100%。
