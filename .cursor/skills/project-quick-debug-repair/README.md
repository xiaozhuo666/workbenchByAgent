## 三个关键特性

### 特性 1：快速诊断

不用你详细描述，我通过快速诊断表就能快速反应：

| 你说 | 我的诊断 | 时间 |
|-----|--------|------|
| "首页样式显示不对" | 定位：样式文件，检查清单：宽度、颜色、间距 | < 30 秒 |
| "待办列表加载不了" | 定位：API + 组件，检查清单：Network、依赖、数据格式 | < 1 分钟 |
| "数据没显示" | 定位：请求层 → 数据层 → 渲染层 | < 1 分钟 |

### 特性 2：精确定位

使用 FILE_MAP，快速找到相关文件，避免全项目扫描：

| 问题类型 | 相关文件数 | 需要读取 | 节省 token |
|---------|---------|---------|---------|
| 首页样式 | 2-3 个 | 50-100 行 | 80% |
| 接口问题 | 3-4 个 | 80-150 行 | 75% |
| 数据问题 | 2-3 个 | 100-150 行 | 70% |

### 特性 3：安全修复

在修复前进行充分诊断，在修复后进行检查，避免引入新问题：

```
诊断阶段：
✅ 问题确认
✅ 原因分析
✅ 影响评估

修复阶段：
✅ 使用 incremental-code-editing（最小化修改）
✅ 检查 React Hooks 依赖
✅ 检查 API 调用正确性

验证阶段：
✅ 功能是否恢复
✅ 有无新问题
✅ 代码逻辑是否正确
```

## 常见问题

### Q1：这个 Skill 能自动修复吗？

A：不能自动修复，但能快速诊断和定位。修复由 incremental-code-editing 完成。

流程：
```
这个 Skill → 诊断 + 定位
incremental-code-editing → 生成修改指令
你 → 应用修改
```

### Q2：如果问题涉及后端怎么办？

A：如果是后端问题（API 返回错误、数据错误等），我会明确指出是后端问题，建议你检查后端代码。

### Q3：这个 Skill 会读取整个项目吗？

A：不会。通过 FILE_MAP 精确定位，只读取相关文件的必要部分。
