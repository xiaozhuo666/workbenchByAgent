# 个人生活助手 — 需求分析文档

**文档类型**：需求分析 (PRD)  
**创建日期**：2026-03-03  
**状态**：Active (Refactoring from Smart Workspace)  
**目标**：将原有的“智能工作台”重构为以 **AI 能力为核心**的“个人生活助手”。通过自然语言交互，集成日程、邮件及生活服务（天气、票务、外卖等）。

---

## 1. 现有功能总结 (Legacy Features)

基于原《智能工作台助手》项目，目前已具备以下核心能力：
- **用户账密体系**：注册、登录、登出、Session 保持、访客模式（MySQL 持久化）。
- **日程/待办管理**：待办项与日程的 CRUD，支持今日/本周视图。
- **QQ 邮箱基础集成**：邮件列表拉取、邮件发送（后端授权管理）。（未实现，暂不处理）
- **基础 AI 对话能力**：接入阿里云百炼（Qwen），支持 SSE 流式输出，具备基础的自然语言理解能力。

---

## 2. 重构计划：个人生活助手定位

### 2.1 首页重构 (Homepage Refactor)
- **目标**：删除原有的“会话中心”独立页面，将首页改为“个人生活助手”聚合页。
- **变更点**：
  - **删除冗余路由**：移除 `/chat` 或 `/conversation` 等独立页面路由及相关冗余代码。
  - **AI 助手常驻化**：AI 助手不再作为一个页面，而是以**侧边栏 (Sidebar)** 或 **浮动对话框** 形式存在于所有页面，随时待命。
  - **生活化看板**：首页突出展示“今日天气”、“今日提醒”、“生活快捷入口”等信息。

### 2.2 冗余代码清理
- 清理原 `Conversation` 页面相关的业务逻辑、样式文件及 API 接口。
- 优化侧边栏导航，仅保留：`首页`、`待办事项`、`日程管理`、`邮件管理`。

## 3. 新增 AI 生活化功能 (P1-P2)

### 3.1 AI 查天气 (Weather Service)
- **描述**：用户在 AI 助手输入“明天上海天气如何？”，助手自动获取并返回天气数据。
- **实现方案**：AI 助手识别天气查询意图 -> 后端/MCP 调用第三方天气 API (如和风天气) -> 结构化返回给用户。

### 3.2 AI 查车票 (Travel/Ticket Service)
- **描述**：用户通过自然语言查询火车/高铁票，如“帮我查一下明天下午北京到上海的高铁”。
- **实现方案**：AI 助手提取城市、时间等参数 -> 调用票务 API 或 MCP 工具查询 -> 展示班次列表。

### 3.3 AI 发 QQ 邮件 (Smart Email)
- **描述**：不再需要手动填写复杂的邮件表单，通过自然语言发起。如：“给张三发封邮件，主题是下周计划，内容是...”
- **实现方案**：
  - AI 助手解析收件人、主题、正文。
  - **MCP/工具调用**：助手调用预定义的 `send_qq_email` 工具。
  - 调用现有后端 QQ 邮箱 SMTP 接口完成发送。

### 3.4 AI 点外卖 (Takeout Service - 最后实现)
- **描述**：AI 助手辅助搜索外卖、推荐店铺。
- **实现方案**：作为进阶功能，主要通过 AI 识别菜品需求，并引导至三方外卖平台或展示推荐列表。

## 5. 技术选型变更
- **AI 助手增强**：引入 **MCP (Model Context Protocol)** 或 **Function Calling** 机制，使助手能直接调用查天气、发邮件的函数。
- **响应式**：强化移动端体验，生活助手需适配手机端的快速操作。
