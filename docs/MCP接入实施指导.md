# MCP 接入实施指导（Node 后端直连方案）

**适用项目**：`ai_project`  
**目标**：指导你使用 spec kit，将「基础 MCP + 12306 MCP + 麦当劳 MCP」分阶段接入现有个人生活助手。  
**原则**：不破坏现有功能，优先查询能力，先灰度后全量。

---

## 1. 先决策：为什么不先上 Dify

当前阶段推荐你采用 **Node 后端直连 MCP**，原因：
- 你已有成熟后端（认证、会话、待办、日程、AI 对话）。
- 接入链路最短，排错最直接。
- 先验证业务价值，再决定是否引入 Dify 做可视化编排。

> 结论：Dify 是可选增强平台，不是 MCP 接入前置条件。

---

## 2. 总体实施路径（建议顺序）

1. **Phase A（基础能力）**：接入基础 MCP（`web_search`、`time_now`）。
2. **Phase B（生活场景）**：接入 12306 MCP（查询）与麦当劳 MCP（查询）。
3. **Phase C（稳定性）**：加超时、重试、熔断、日志看板。

每个 Phase 都要满足：
- 有独立开关（Feature Flag）。
- 有降级路径（工具失败 -> 纯模型回答）。
- 有验收用例（至少 5 条）。

---

## 3. 你需要准备的输入信息（给 spec kit）

在执行前先整理以下信息，避免中途反复：

### 3.1 MCP Server 清单
- Server 名称（例：`base-tools`、`mcp-12306`、`mcp-mcd`）
- 传输方式（`stdio` 或 `http`）
- 启动方式（命令、镜像或 URL）
- 可用工具列表（工具名 + 参数 schema）

### 3.2 安全与凭据
- 每个 Server 所需的 API Key / Token
- 凭据注入方式（环境变量名）
- 允许调用的工具白名单

### 3.3 业务约束
- 12306、麦当劳只做“查询”，不做“下单/支付”
- 高风险写操作默认禁用
- 回答必须中文，且给出关键字段摘要

---

## 4. spec kit 执行建议（可直接照抄）

你后续可按下面 3 轮任务喂给 spec kit。

### 4.1 任务一：MCP 基础框架
**目标**：在后端增加 MCP Tool Adapter，不改变现有接口行为。  
**关键要求**：
- 新增工具注册中心与执行器。
- 聊天接口支持工具调用循环（最多 N 轮，防止死循环）。
- 工具调用失败自动降级。
- 增加调用日志表或日志字段（会话ID、工具名、耗时、状态）。

### 4.2 任务二：接入 12306 MCP
**目标**：支持“车次/余票查询”自然语言问答。  
**关键要求**：
- 只暴露查询工具。
- 输出结构化字段：车次、发车时间、到达时间、耗时、余票概览。
- 参数缺失时先追问（如日期、出发地）。

### 4.3 任务三：接入麦当劳 MCP
**目标**：支持门店、菜单、活动信息查询。  
**关键要求**：
- 只暴露查询工具。
- 输出结构化字段：门店名、距离（如有）、营业时间、菜单价格区间、活动信息。
- 信息不确定时明确标注“以官方页面为准”。

## 5. 推荐目录与模块边界（实现时参考）

- `backend/src/modules/ai/`：
  - `tool-registry`（工具注册）
  - `tool-executor`（执行与超时/重试）
  - `mcp-adapter`（与 MCP Server 通信）
  - `ai.service`（模型交互与工具循环）
- `backend/src/config/`：
  - MCP 相关环境变量读取与校验
- `docs/`：
  - 记录工具清单、版本、验收结果

## 6. 测试与验收（必须做）

### 6.1 回归测试（防止影响原功能）
- 登录/注册/鉴权
- 待办 CRUD
- 日程 CRUD
- 普通聊天（不触发工具）

### 6.2 MCP 功能测试
- 基础工具查询成功
- 12306 查询成功
- 麦当劳查询成功
- 工具超时/异常后可降级
- 非白名单工具调用被拦截

### 6.3 上线门槛
- 工具成功率 >= 95%（灰度环境）
- 工具平均耗时 <= 3s（不含第三方极端抖动）
- 无 P0 回归问题

---

## 7. 生产部署注意事项（Docker 场景）

- 采用环境变量管理 MCP 配置，不硬编码密钥。
- 每个 MCP Server 单独健康检查。
- 外网依赖建议设置超时和重试上限。
- 建议先灰度一小部分流量，再全量放开。

---

## 8. 常见问题排查

### Q1：工具调用一直不触发
- 检查模型请求是否携带了 tools 定义。
- 检查工具描述是否足够明确（何时调用、输入参数）。

### Q2：调用成功但回答质量差
- 增强系统提示词：要求先工具后总结，禁止臆测。
- 标准化工具返回格式，减少自由文本噪声。

### Q3：新增 MCP 后原功能异常
- 立即关闭对应 Feature Flag。
- 回滚到纯模型路径，先保障主流程可用。
